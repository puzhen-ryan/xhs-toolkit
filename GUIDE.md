## 一、排版规范

### 1.1 尺寸与格式
- **固定尺寸：1080 × 1920px（9:16竖屏）** — 手机全屏沉浸
- 输出格式：JPG，quality 80+
- ❌ 不要用 1080×1440（3:4），手机上下留白太多

### 1.2 字体
- **正文字体：Noto Sans SC**（已验证 headless 可渲染）
- 标题 font-weight: **900**（Black），正文 **700**（Bold）
- ❌ 得意黑装了但 headless Chromium 不加载
- ❌ Google Fonts @import 会导致 headless 卡死（无网络）

### 1.3 字号规范（手机可读性优先）
| 元素 | 字号 | 说明 |
|------|------|------|
| 封面大标题 | 56-110px | 越大越抢眼，超大艺术字更好 |
| 页内标题 | 42-56px | 加粗，每页最醒目的文字 |
| 副标题/区块标题 | 28-34px | 颜色高亮，weight 800 |
| 正文内容 | 24-28px | 不低于24px！手机上22px已经很费劲 |
| 标签/badge | 18-22px | 辅助信息，可稍小 |
| 页脚水印 | 20-22px | 低透明度，不抢视觉 |

> ⚠️ 大哥反馈过"字体要大要粗，手机看不清"——**宁大勿小**

### 1.4 信息密度（最重要的原则）
- **每张 slide 内容要填满，不留大块空白**
- 上中下三段式布局：标题区 → 主内容 → 评论/总结
- 内容不够就加：对比表、数据卡片、标签云、时间线、赛道全景
- 每页底部放一句"Bear点评"或"一句话总结"，既填充又增加观点输出
- 页脚统一水印：`小红书 @朴真 · Bear 🐻 | 系列名`

### 1.5 配色方案
采用**浅色笔记风**，符合小红书平台主流审美：

| 元素 | 色值 | 说明 |
|------|------|------|
| 📄 页面背景 | #FDF8F3 | 暖白/米白，有纸张质感 |
| 🗂️ 卡片背景 | #FFFFFF | 纯白，配合轻阴影(0 2px 12px rgba(0,0,0,0.06)) |
| 📝 主文字 | #1a1a2e | 深灰近黑，不用纯黑 |
| 📝 辅助文字 | #888 / #999 | 浅灰，用于描述/编号 |
| 🟠 橙色（主色） | #FF6B35 | 标题强调/标签/翻车争议 |
| 🔵 蓝色 | #4A90D9 | 产品发布/技术/冷静分析 |
| 🟡 金色 | #F5A623 | 融资/金额/重点数据 |
| 🟢 绿色 | #7ED321 | 新发布/正面/成功 |
| 🟣 紫色 | #9B59B6 | 商业化/创新/差异化 |
| 分隔线 | #eee | 底部datebar上方 |

> ⚠️ 暗色底(#0a0a0a)在小红书不是主流，浅色笔记风更符合平台调性和用户习惯

### 1.6 视觉组件库
常用信息展示组件（直接从以往帖子复用）：

- **数据卡片网格**：2×2 或 2×3 grid，大数字+小标签
- **VS对比框**：左右双栏 + 中间VS字样
- **时间线**：左侧圆点 + 文字描述，适合事件回顾
- **标签云**：flex wrap 圆角tag，展示技术栈/产品列表
- **表格**：header行高亮，当前行用accent border标记
- **引用/评论块**：左边框4px高亮色 + 半透明背景
- **赛道全景图**：分国产/海外两行标签，展示行业格局
- **数字hero**：超大字号数字（80px+）居中，下方小字说明

### 1.7 CSS 模板结构
```css
* { margin:0; padding:0; box-sizing:border-box; }
body { width:1080px; height:1920px; background:#FDF8F3; 
       font-family:'Noto Sans SC',sans-serif; overflow:hidden; }
.page { padding:60px; height:100%; display:flex; flex-direction:column; }
/* 卡片：白底 + 圆角 + 轻阴影 */
.card { background:#fff; border-radius:16px; padding:28px 30px;
        box-shadow:0 2px 12px rgba(0,0,0,0.06); }
/* 左侧彩色边框区分不同条目 */
.card-accent { border-left:5px solid #FF6B35; }
/* 页脚用 margin-top:auto 贴底 */
```

## 二、内容规范

### 2.1 选题原则
**优先级排序：**
1. 🔥 **今日热点**（翻车/争议 > 产品发布 > 融资）— 时效性最强
2. 📊 **横评对比**（多产品PK，如"国产AI编程大战"）— 信息量大
3. 📰 **周报合集**（AI周报 Vol.xx）— 一次覆盖多条新闻
4. 📝 **论文解读**（arXiv热门论文拆解）— 专业深度
5. 🛠️ **工具教程**（使用方法/技巧）— 实用长尾

> 热点事件优先，争议性内容流量最好

### 2.2 标题规范
- ⚠️ **标题必须 ≤ 20字！** 超过20字发布按钮会被禁用，但无明显报错
- 结构：`核心事件 + 情绪钩子`
- 好标题示例：
  - ✅ `AI周报Vol.01：本周五大事件`（14字）
  - ✅ `Step 3.5 Flash：小参数大智慧`（14字）
  - ✅ `6000位CEO承认AI没用`（10字）
  - ✅ `Reasoning模型的5种崩溃`（11字）
- ❌ 避免：太长、太学术、无情绪

### 2.3 正文文案结构
```
🔥 一句话hook（为什么要看这篇）

[逐条新闻/知识点摘要，带emoji编号]

💡 Bear的观点（个人见解，有态度）

#标签1 #标签2 #标签3 ...（8-12个）
```

### 2.4 标签（Hashtag）策略
- 每篇 8-12 个标签
- 结构：大流量通用标签（#AI #人工智能）+ 具体话题标签（#GLM5 #Qwen）+ 系列标签（#AI周报）
- 标签放在正文最后

### 2.5 Slides 内容结构（推荐）
| Slide | 内容 | 要点 |
|-------|------|------|
| #1 封面 | 超大标题 + 副标题 + 目录/预览 | 第一印象决定点击率 |
| #2-N 内容页 | 每页一个核心信息点 | 信息密度要高，不留大白 |
| 最后一页 | Bear观点/总结 + 引导关注 | 个人品牌输出 |

### 2.6 内容调性
- **有态度**：不做纯搬运，每条新闻/知识点附加Bear评论
- **敢下判断**：如"AI中立性正在崩塌"、"产品力 > 发布速度"
- **适度调侃**：如"程序员的钱包要遭殃了"、"巧合？我觉得不是"
- **专业但不晦涩**：技术名词首次出现时用通俗语言解释

## 三、发布流程

### 3.1 制作流程
1. 选题 → 调研（web_fetch 抓新闻/论文）
2. 写 HTML slides → 用 OpenClaw `write` 工具（❌ 禁止用 PowerShell Set-Content，会破坏UTF-8编码）
3. Playwright 截图 → **逐张跑**，循环超过5张会crash
4. 发 Telegram 给大哥预览
5. 大哥确认 → 运行 publish.js 发布

### 3.2 发布脚本要点
- 先点"上传图文"tab（默认停在"上传视频"）
- 图片逐张 uploadFile（input不支持multiple）
- 标题填入 `#publishInput`，正文填入 `#post-textarea` 或 `.ql-editor`
- 发布成功标志：URL含 `publish/success` 或 `published=true`
- 失败标志：URL含 `from=tab_switch`

### 3.3 技术环境
- 浏览器：**Edge**（非Chrome），路径 `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`
- CDP端口：`--remote-debugging-port=9222`
- Playwright截图：headless Chromium launch，1080×1920 viewport
- 截图前清理残留 file:// 标签页（之前 kill 的进程没关页面）

## 四、踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 标题超20字发布失败 | 小红书标题限制20字，无明显报错 | 标题严格≤20字 |
| PowerShell写中文HTML乱码 | Set-Content 默认GBK编码 | 用OpenClaw `write` 工具 |
| Google Fonts卡死 | headless无网络 | 不用@import，用本地Noto Sans SC |
| Playwright截图crash | 循环跑5张+会内存溢出 | 逐张跑snap.js |
| 得意黑不渲染 | headless Chromium不加载新安装字体 | 接受Noto Sans SC |
| 发布按钮点不到 | Puppeteer `.click()` 报"outside viewport" | 用 `element.evaluate(el => el.click())` |
| 春节祝福帖生肖错 | 2025=蛇年，2026=马年 | 发之前核实年份对应生肖 |
| 默认停在"上传视频"tab | Creator页面默认视频tab | 脚本先点"上传图文" |
